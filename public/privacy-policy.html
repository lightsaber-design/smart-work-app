Política de Privacidad

MinistryLog · Última actualización: 7 de julio de 2026

1. Responsable de los datos

Esta aplicación es desarrollada de forma individual por Marco Antonio Tévar Asensio. Contacto: carol.torres.dev2@gmail.com

2. Datos que la app recopila

MinistryLog no recopila ni transmite datos personales a ningún servidor propio. Toda la información que el usuario introduce se almacena únicamente en el dispositivo (y, si el usuario lo activa explícitamente, en su propia cuenta de Google Drive; ver sección 7):

  • Nombre del usuario (opcional, solo para personalización local)
  • Ciudad de referencia (nombre, latitud y longitud) para cálculos de horario solar
  • Registros de tiempo y actividades de ministerio
  • Eventos de calendario y sesiones de estudios bíblicos
  • Notas de voz grabadas por el usuario tras un estudio (opcional, ver sección 4)
  • Lugares favoritos marcados en el mapa
  • Preferencias de configuración (idioma, tema, notificaciones)
  • Si se inicia sesión con Google: nombre y correo de la cuenta, solo para identificarla (ver sección 7)

3. Permiso de ubicación

La app solicita acceso a la ubicación del dispositivo únicamente para:

  • Autocompletar la selección de ciudad durante la configuración inicial.
  • Centrar el mapa al añadir un lugar favorito.

Las coordenadas de ubicación nunca se envían fuera del dispositivo. El usuario puede conceder o revocar este permiso en cualquier momento desde Ajustes → Aplicaciones → MinistryLog → Permisos, o desde Ajustes → Permisos dentro de la propia app.

4. Permiso de micrófono

La app solicita acceso al micrófono únicamente para grabar una nota de voz opcional justo después de completar un estudio bíblico. La grabación se guarda solo en el dispositivo, nunca se transmite a ningún servidor, y el usuario puede eliminarla en cualquier momento. Este permiso puede revocarse igual que el de ubicación.

5. Conexión a Internet

La app utiliza Internet para:

  • Cargar los mapas de calles (teselas de OpenStreetMap, proveedor: openstreetmap.org).
  • Consultar el tiempo meteorológico de la ciudad configurada (proveedor de datos abiertos, sin identificar al usuario).
  • Si el usuario lo activa: iniciar sesión con Google y sincronizar la copia de seguridad con su Google Drive (ver sección 7).

Estas solicitudes no incluyen datos personales del usuario más allá de lo estrictamente necesario para la funcionalidad solicitada. El resto de la funcionalidad de la app es completamente offline.

6. Notificaciones

Las notificaciones locales (recordatorio de temporizador, objetivo mensual, estudios pendientes) se generan íntegramente en el dispositivo mediante el sistema de notificaciones de Android. No se utilizan servicios de push de terceros.

7. Copias de seguridad, exportación y Google Drive

El usuario puede exportar todos sus datos en formato JSON en cualquier momento desde Ajustes → Copia de seguridad. Estos ficheros permanecen bajo el control exclusivo del usuario.

Adicionalmente, MinistryLog ofrece la opción de iniciar sesión con una cuenta de Google para guardar una copia de seguridad de estos mismos datos en Google Drive. Esta función es opcional y solo se activa si el usuario la solicita explícitamente. Los detalles:

  • Se solicita el permiso de Google drive.appdata, que da acceso únicamente a una carpeta oculta y privada de esta app dentro de la cuenta de Drive del usuario — la app no puede ver, leer ni modificar ningún otro archivo o carpeta del Drive del usuario.
  • Se solicitan también los permisos básicos de perfil y correo electrónico únicamente para mostrar qué cuenta está conectada.
  • Ningún dato pasa por servidores propios de MinistryLog: la copia de seguridad va directamente del dispositivo del usuario a su propia cuenta de Google Drive.
  • El usuario puede cerrar sesión y desconectar esta función en cualquier momento desde Ajustes → Copia de seguridad en Google Drive.

La copia de seguridad automática nativa de Android está desactivada para esta app, de modo que los datos no se suben a ningún sitio sin la acción explícita descrita arriba.

8. Datos de terceros

MinistryLog no integra SDKs de analítica, publicidad ni redes sociales, y no usa ningún servicio de seguimiento de usuarios. La única integración de terceros es el inicio de sesión de Google descrito en la sección 7, usado exclusivamente para esa función y bajo el control directo del usuario.

9. Menores de edad

La app no está dirigida a menores de 13 años. No se recopilan datos de forma intencionada de ningún usuario.

10. Cambios en esta política

Cualquier modificación se publicará en esta misma página. El uso continuado de la app tras la publicación de cambios implica la aceptación de los mismos.

11. Contacto

Para cualquier consulta relacionada con la privacidad, escribe a: carol.torres.dev2@gmail.com

Privacy Policy

MinistryLog · Last updated: July 7, 2026

1. Data Controller

This app is developed individually by Marco Antonio Tévar Asensio. Contact: carol.torres.dev2@gmail.com

2. What data the app collects

MinistryLog does not collect or transmit personal data to any server we operate. All information entered by the user is stored on-device only (and, only if the user explicitly enables it, in the user's own Google Drive account; see section 7):

  • User name (optional, for local personalization only)
  • Reference city (name, latitude, longitude) for solar schedule calculations
  • Time entries and ministry activity records
  • Calendar events and Bible study sessions
  • Voice notes recorded by the user after a study (optional, see section 4)
  • Favorite places marked on the map
  • App preferences (language, theme, notifications)
  • If the user signs in with Google: the account's name and email, solely to identify it (see section 7)

3. Location permission

The app requests device location access only to:

  • Auto-fill the city selection during initial setup.
  • Center the map when adding a favorite place.

Location coordinates are never sent outside the device. This permission can be granted or revoked at any time via Settings → Apps → MinistryLog → Permissions, or from Settings → Permissions inside the app.

4. Microphone permission

The app requests microphone access only to record an optional voice note right after completing a Bible study. The recording is stored on-device only, is never transmitted to any server, and the user can delete it at any time. This permission can be revoked the same way as location.

5. Internet access

The app uses the Internet to:

  • Load street map tiles (OpenStreetMap, provider: openstreetmap.org).
  • Fetch weather data for the configured city (open weather data provider, no user identification involved).
  • If enabled by the user: sign in with Google and sync the backup to the user's Google Drive (see section 7).

These requests do not include personal user data beyond what's strictly necessary for the requested feature. All other app functionality works fully offline.

6. Notifications

Local notifications (timer reminder, monthly goal, pending studies) are generated entirely on-device using the Android notification system. No third-party push services are used.

7. Backup, export, and Google Drive

Users can export all their data as a JSON file at any time via Settings → Backup. These files remain under the user's exclusive control.

MinistryLog additionally offers the option to sign in with a Google account to back up this same data to Google Drive. This feature is optional and only activates if the user explicitly requests it. Details:

  • The app requests Google's drive.appdata permission, which grants access only to a hidden, app-private folder within the user's Drive account — the app cannot see, read, or modify any other file or folder in the user's Drive.
  • Basic profile and email permissions are also requested solely to display which account is connected.
  • No data passes through any server operated by MinistryLog: the backup goes directly from the user's device to their own Google Drive account.
  • The user can sign out and disconnect this feature at any time via Settings → Google Drive Backup.

Android's native automatic backup is disabled for this app, so data is not uploaded anywhere without the explicit action described above.

8. Third-party data

MinistryLog does not include any analytics, advertising, or social media SDKs, and does not use any user-tracking service. The only third-party integration is the Google sign-in described in section 7, used exclusively for that feature and fully under the user's control.

9. Children

This app is not directed at children under 13. No data is intentionally collected from any user.

10. Changes to this policy

Any changes will be published on this page. Continued use of the app after changes are posted constitutes acceptance of those changes.

11. Contact

For any privacy-related questions, write to: carol.torres.dev2@gmail.com
